# RawHID Host アプリ操作マニュアル

## 共通操作

左側の Sidebar から画面を切り替えます。Sidebar 下部の `JP / EN` で日本語 / 英語表示を切り替えられます。

`保存` ボタンがあるページでは、画面上で変更した内容を保存します。保存に成功した場合は特別なメッセージを出しません。保存に失敗した場合だけエラーを表示します。

Dashboard のトグルは即時保存です。保存に成功すると「✓ 保存しました」が短く表示されます。保存に失敗した場合は、トグル表示が操作前の状態に戻ります。

Layer Rules は自動保存です。ルールの追加、削除、変更を行った時点で保存されます。

## Dashboard

![Dashboard](images/manual-dashboard.svg)

Dashboard は、アプリ全体の状態を確認し、監視を開始 / 停止する画面です。

`監視開始` を押すと、Raw HID 対応キーボードの監視を開始します。監視中は、前面アプリに応じたレイヤー切り替え、時刻同期、AI 使用量送信などが有効になります。

`監視停止` を押すと、キーボードへの送信を停止します。AI 使用量の取得など、キーボード送信とは独立して動く機能は、画面側で更新できる場合があります。

Dashboard では次の情報を確認できます。

- 監視中 / 停止中
- 接続デバイス数
- レイヤールールの有効 / 無効
- 時刻同期の有効 / 無効
- AI 使用量の有効 / 無効
- AI 使用量の簡易サマリ
- 直近ログ

AI Usage 全体、Codex、Claude Code のトグルも Dashboard から切り替えられます。詳細な状態確認や細かい設定は、Sidebar の `AI 使用量` 画面で行います。

## レイヤールール

![Layer Rules](images/manual-layer-rules.svg)

レイヤールールでは、前面にあるアプリに応じて、キーボード側のレイヤーを切り替えるルールを作成できます。

左側に現在起動中のアプリ一覧が表示されます。各アプリには、実行ファイルから取得した実際のアプリアイコンが表示されます。アイコンを取得できないアプリは、アプリ名の頭文字で表示されます。

対象アプリを選び、切り替えたいレイヤーを選択して `追加` を押すと、そのアプリ用のルールが作られます。作成済みのルールカードにもアプリアイコンが表示されます。

レイヤールールは**キーボードごと**に設定します。全キーボード共通のルールはありません。キーボードによってレイヤー構成が異なるためです。

画面右上のデバイス選択で、ルールを編集するキーボードを選びます。

- 一覧には、接続中のキーボードと、過去にルールを設定したキーボードが表示されます。
- 未接続のキーボードには `(未接続)` が付きます。切断中でもルールの編集は可能です。
- ルールを設定していないキーボードは、レイヤー切り替えの対象になりません。

デバイス選択の隣のゴミ箱ボタンで、選択中キーボードの設定 (ルール一式) を削除できます。削除したキーボードはレイヤー切り替えの対象外に戻ります。

アプリがルールに一致しなくなった場合は、設定された未一致時の動作に従います。通常はキーボード側のアプリレイヤーを解除します。デバイス専用設定に未一致時の動作が指定されている場合は、既定値よりもデバイス側の指定が優先されます。

監視中は、前面アプリの切り替わりを即時に検知してレイヤーを切り替えます。ポーリング間隔を待つ必要はありません。

## アクション

アクション画面では、キーボードのキーから PC 側の操作を実行するバインディングを設定します。キーボード側で `&host_action <ID> <値>` を割り当てたキーを押すと、HOST_ACTION パケットが PC に届き、対応する動作が実行されます。

キーボード側 (ZMK firmware) で `&host_action` をキーに割り当てる手順 (モジュールの取り込み、`CONFIG_RAWHID_APP_HOST_ACTION=y`、keymap への記述例) は、`zmk-rawhid-app` の README「Host action uplink」セクションを参照してください。ここで決めた `<ID>` と、本画面で設定する `アクションID` を一致させる必要があります。

この機能は HOST_ACTION 対応キーボード（capability を持つデバイス）でのみ使えます。対応キーボードがない場合は、画面右上に「HOST_ACTION 対応キーボードがまだありません」と表示されます。

画面右上では次の操作ができます。

- 編集対象デバイスの選択（過去に設定したデバイスは未接続でも表示され、`(未接続)` が付きます）
- ゴミ箱ボタンで、選択中デバイスのバインディング一式を削除
- `アクション実行` トグルで、アクション実行全体の有効 / 無効を切り替え

アクション実行はセキュリティのため**既定で無効**です。無効のままでは、キーボードから HOST_ACTION が届いてもログに記録されるだけで実行されません。右上のトグルで有効にします。

バインディングは次の手順で追加します。

1. `アクションID`（0〜255）を入力します。キーボード側 `&host_action <ID> <値>` の `<ID>` と一致させます。追加時は、そのデバイスでまだ使われていない最小のIDが初期値として入ります。
2. `動作` を選びます。選べる動作は次のとおりです。
   - `このアプリの画面を開く`: RawHID Host のウィンドウを前面に表示します。最小化されているときやシステムトレイに隠れているときも復帰して前面に出ます。
   - `監視を停止`: 監視を停止します。
   - `AI 使用量を更新`: AI 使用量を再取得します。
   - `アプリを起動`: 指定した実行ファイルを起動します。
   - `フォルダを開く`: 指定したフォルダを Windows Explorer で開きます。すでにそのフォルダが開かれていれば、そのウィンドウを前面に表示します。開かれていなければ新しいウィンドウで開きます（Explorer が起動していない場合は起動します）。
3. `アプリを起動` または `フォルダを開く` を選んだ場合は、`実行ファイル` / `フォルダ` にパスを入力します（例: `C:\tools\example.exe` / `C:\Users\me\Documents`）。
   - `フォルダを開く` では `既存ウィンドウのタブで開く（実験的）` トグルを選べます。オンにすると、新しいウィンドウではなく既存の Explorer ウィンドウの新規タブで開こうとします。タブを開く公開 API が無いためベストエフォートで、OS バージョンや状況により失敗することがあります（その場合は新しいウィンドウで開きます）。
4. `追加` を押します。すでに同じアクションIDがある場合や、パスが必要な動作でパス未指定の場合はエラーになります。エラーは入力欄を直し始めると自動的に消えます。

設定済みのバインディングは一覧に表示されます。一覧は**アクションID順**に並びます。

- パスを持つ動作（`アプリを起動` / `フォルダを開く`）の行には鉛筆ボタンが表示され、押すと上のフォームに内容が読み込まれます。値を変更して `更新` を押すと、そのバインディングが書き換わります（削除して追加し直す必要はありません）。`キャンセル` で編集をやめると追加モードに戻ります。
- パスを持たない動作（`このアプリの画面を開く` など）は編集する項目がないため鉛筆ボタンは表示されません。変更したい場合は削除して追加し直します。
- ゴミ箱ボタンでバインディングを削除できます。

アクションはキーボードに割り当てた `<値>` を path やコマンドとして解釈しません。実行内容はこの画面で設定したものだけです。

アクションは**監視中のみ**実行されます。監視を停止している間に届いた HOST_ACTION は実行されません。

## 時刻同期

![Time Sync](images/manual-time-sync.svg)

時刻同期では、PC の時刻情報をキーボードへ送信できます。キーボード側が TIME_SYNC 表示に対応している場合、キーボードの画面や表示部に時刻を出せます。

画面では次の操作ができます。

- 時刻同期の有効 / 無効
- 表示形式の選択
- 12時間 / 24時間表示の選択
- 同期間隔の変更
- タイムゾーンの選択 (「自動」= PC の設定、または `UTC±hh:mm` のプリセット)

変更したら `保存` を押します。監視中の場合、保存後の設定は runner に反映され、以後の送信に使われます。

## AI 使用量

![AI Usage](images/manual-ai-usage.svg)

AI 使用量では、Codex と Claude Code の 5h / 7d 使用率を取得し、対応キーボードへ送信できます。

画面では次の操作ができます。

- AI Usage 全体の有効 / 無効
- Codex の有効 / 無効
- Claude Code の有効 / 無効
- 取得間隔の変更
- stale 判定秒数の変更
- 使用率の確認
- `更新` による手動更新
- Advanced で詳細設定の確認 / 編集

`更新` を押すと、AI 使用量の再取得を background worker に要求します。取得処理は画面操作とは別に裏側で行われます。連打しても多重取得しないよう制御されます。

Codex の使用量は、複数の sessions ディレクトリをまとめて読み取れます。`Advanced` の `sessions 自動検出` を有効にすると、`Codex sessions dir` に加えて Windows の既定パス・各 WSL ディストロの `~/.codex/sessions`・`追加 sessions paths` を読み込み、合算した使用量を表示します。WSL 上の Codex CLI を使っている場合も、`WSL sessions を含める` を有効にすればその使用分が反映されます。

使用率バーは「使用済み」の割合です。80% 以上でオレンジ、90% 以上で赤になります。値が取れていない場合は `no data` と表示されます。

### AI Usage の状態表示

AI Usage では provider ごとに状態が表示されます。

| 表示 | 意味 | ユーザーが見るべきこと |
| --- | --- | --- |
| `ok` | 最新の使用量を取得できています。 | 通常状態です。 |
| `stale` | 前回取得した値を表示していますが、一定時間以上更新できていません。 | 表示値は古い可能性があります。次回更新や `更新` を試してください。 |
| `no data` | まだ使用量として使えるデータがありません。 | Codex / Claude Code を使用した後に更新してください。 |
| `missing credentials` | Claude Code の認証情報が見つかりません。 | Claude Code でログインしている環境を確認してください。 |
| `expired credentials` | 認証情報はありますが期限切れです。 | Claude Code のログイン状態を更新してください。 |
| `auth failed` | 認証情報で API にアクセスできませんでした。 | ログインし直す、または別環境の認証情報を確認してください。 |
| `rate limited` | API 側で一時的に制限されています。 | 少し待ってから再試行してください。 |
| `fetch failed` | API 通信または取得処理に失敗しました。 | ネットワークや Claude Code 側の状態を確認してください。 |
| `parse failed` | 取得した情報の形式を解釈できませんでした。 | schema 変更や未対応形式の可能性があります。 |
| `disabled` | provider が無効です。 | 必要ならトグルを有効にしてください。 |

`stale` は「完全に失敗」ではありません。過去に成功した値を保持したまま、現在は新しい値を取得できていない状態です。キーボード側へ送る packet にも stale 情報が含まれます。

### Source 表示

`Source` は、その使用率がどこから来たかを表します。

| Source | 意味 |
| --- | --- |
| `Codex rate_limits` | Codex の local history から rate_limits を取得できた状態です。quota 情報として扱えます。 |
| `Codex local history estimate` | rate_limits が取れず、履歴から activity estimate を出している状態です。実 quota ではありません。reset 時刻は表示しません。 |
| `Claude OAuth API` | Claude Code の OAuth usage API から取得できた状態です。experimental / best-effort として扱います。 |
| `source なし` | 使用量の取得元として使える情報がまだありません。 |

Claude Code の access token、credentials JSON、API response、raw error は画面に表示しません。

## キーマップ表示

![Keymap Viewer](images/manual-keymap-viewer.svg)

キーマップ表示では、ZMK Studio 対応キーボードの現在のキーマップを表示専用で確認できます。

左側には Studio 対応デバイスだけが表示されます。デバイスを選択すると、そのキーボードのキーマップを自動で読み取ります。

画面では次のことができます。

- Studio 対応デバイスの選択
- レイヤー一覧の確認
- レイヤーごとのキー割り当て確認
- physical layout に近いキー配置での表示
- physical layout が取れない場合のグリッド表示

編集、保存、書き込み、unlock 操作は行いません。Studio が locked の場合は、キーマップを読み取らず、キーボード側で unlock が必要であることを表示します。

キー右下の `#0`、`#1` などは key position です。ZMK の keymap 上で、そのキーが何番目の position かを表します。

監視中、キーボード側でレイヤーを手動で切り替えると、対応するレイヤーボタンの右上にオレンジの ● が付きます (LAYER_STATE 対応キーボードのみ)。

### ヒートマップ

「ヒートマップ」タブでは、キーごとの打鍵数を物理レイアウト上に色分け表示します (KEY_STATS 対応キーボードのみ)。

- 期間フィルタ: 今日 / 7日間 / 全期間
- 総打鍵数、TOP 5 キー、左右バランスを表示
- 色は打鍵数が多いほど 青 → オレンジ → 赤 になります
- 記録されるのは「キー位置ごとの回数」だけです。何を入力したか (キーの内容) は記録も送信もされません

統計はキーボードの uid ごとにローカルファイルへ保存されます。Host Link デバイスとシリアル番号が一致しない場合、ヒートマップは表示できません (記録は継続されます)。

## デバイス

![Devices](images/manual-devices.svg)

デバイス画面では、Host Link と ZMK Studio の対応状態を確認できます。

Host Link は、RawHID Host 独自の packet protocol に対応しているかを表します。`OK` のデバイスは、AI_USAGE、TIME_SYNC、APP_LAYER などの送信対象になります。

ZMK Studio は、ZMK Studio RPC に対応しているかを表します。Keymap Viewer では、この Studio 対応デバイスを使ってキーマップを読み取ります。

Host Link 対応デバイスと Studio 対応デバイスは、必ずしも同じとは限りません。同じキーボードが両方に対応する場合もあります。

BATTERY 対応キーボードを監視中は、デバイスカードに本体 / 左 / 右のバッテリー残量が表示されます。残量はキーボード側からの報告で更新され、未接続のペリフェラルは `--` になります。Dashboard の接続デバイス欄にも同じ残量が表示されます。

## 設定

![Settings](images/manual-settings.svg)

設定画面では、アプリ全体の基本設定を変更できます。

### 外観

- `アクセント色`: UI 全体の強調色 (トグル、選択状態、ステータスランプなど) を変更します。プリセット 6 色から選ぶか、`+` を押して OS のカラーピッカーから独自の色を追加できます (最大 8 件)。
- カスタム色は、スウォッチにマウスを乗せると表示される × で削除できます。使用中の色は削除できません。
- 選択は即時反映・即時保存されます。この設定はこの PC の UI にのみ保存され、設定ファイル (`rawhid-host.toml`) やキーボードには影響しません。

### アプリの起動

- `起動時に監視を開始`: アプリを起動したとき、自動で監視を開始します。
- `Windows ログイン時に起動`: Windows へのログイン時に RawHID Host を自動起動します。このトグルは操作した時点で Windows のスタートアップ登録 (レジストリ) に反映されます。登録に失敗した場合は、トグル表示が元の状態に戻ります。

2 つを組み合わせると、PC を起動するだけで監視まで自動で始まる構成にできます。

### 基本設定

`更新` を押すと、保存済みの設定を読み直します。画面上の未保存変更がある場合は、保存済み設定で上書きされます。

`保存` を押すと、現在の画面内容を設定ファイルへ保存します。成功時は何も表示しません。失敗時だけエラーを表示します。

HID の Usage Page / Usage は 16 進数で入力します。`0x0000`〜`0xFFFF` の範囲外や解釈できない値はエラーとして表示され、保存できません。

通常の操作では、細かい設定ファイルの場所や中身を意識する必要はありません。トラブルシュートや詳細調整が必要な場合だけ、設定ファイルを直接確認します。

## システムトレイ

ウィンドウを閉じても、アプリは完全終了せずシステムトレイに残ります。

トレイメニューからは次の操作ができます。

- `Start monitoring` / `Stop monitoring`: ウィンドウを開かずに監視を開始 / 停止します。
- `Show window`: ウィンドウを再表示します。最小化やトレイ常駐の状態からも復帰します。
- `Quit`: アプリを完全に終了します。

トレイアイコンを左クリックすると、ウィンドウの表示 / 非表示を切り替えられます。

BATTERY 対応キーボードを監視中は、トレイアイコンにマウスを乗せると、ツールチップにバッテリー残量が表示されます。デバイスごとに 1 行で表示し、左右に分かれるキーボードは `L 90% / R 88%` のように表示します。監視を停止すると `RawHID Host` の表示に戻ります。

アプリはシングルインスタンスです。すでに起動している状態でもう一度起動した場合は、新しいウィンドウは開かず、既存のウィンドウが前面に表示されます（最小化やトレイ常駐からも復帰します）。
